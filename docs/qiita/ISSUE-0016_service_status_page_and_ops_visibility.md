# ISSUE-0016 サービス状況表示ページと運用ステータス可視化（下書き）

## 概要
QuizVerse に公開向け `/status` ページと、運用向けステータスAPIを追加した。MVPとして、システム内で取得できる情報を使い、最新状態のみを表示する。

## 背景
- 利用者が「今使えるか」を判断できる導線が不足していた
- 管理画面ではメール設定など個別機能はあるが、横断的な稼働可視化がなかった

## 実装内容
- Backend
  - `GET /api/status` を追加
  - `GET /api/admin/status` を追加（仮置き `X-Admin-Mode: true` 判定）
  - `application / api / database / auth / mail` を判定
  - `maintenance` 情報を環境変数から取得
- Frontend
  - `/status` 公開ページを追加
  - 状態カード、ステータスバッジ、更新時刻表示
  - skeleton loading / 空状態 / エラー状態を実装
  - ダーク/ライト双方で可読性を維持

## 状態設計上の工夫
- 判定は `normal / warning / outage / maintenance` に統一
- DBは重いクエリではなく `SELECT 1` の軽量疎通
- メールは実送信確認ではなく設定妥当性で判定し、MVPコストを抑制
- メンテナンス状態は環境変数で即時切替可能にして運用性を担保

## 注意点
- `health = 品質保証` ではないため、ユーザー体験との乖離には別途監視が必要
- 公開ページに内部情報を出しすぎないよう、公開APIと管理APIを分離
- 現状は最新状態のみ（履歴は未実装）
