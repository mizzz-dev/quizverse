# ISSUE-0016: サービス状況表示ページと運用ステータス可視化

## 背景
運用中の QuizVerse で、利用者が「今使えるか」を判断できる表示が不足している。管理側でも主要コンポーネントの状態把握を単一画面で行える基盤が必要。

## 目的
- 一般ユーザーがサービス利用可否を確認できる公開ステータスページを提供する。
- 管理者が内部向け補足情報を取得できるAPIを提供する。
- MVPとして、最新状態のみを簡易判定で返す。

## 対応範囲
- `GET /api/status` の追加（公開向け）
- `GET /api/admin/status` の追加（仮置き管理権限向け）
- `/status` ページ（ステータスカード・状態バッジ・更新時刻・メンテナンス告知）
- ローディング/空状態/エラー状態の整備
- README / docs / Qiita下書きの更新

## 状態判定（MVP仮置き）
- application/api: アプリ稼働中を `normal`
- database: `SELECT 1` による軽量疎通確認
- auth: JWT設定有無を基準に判定
- mail: SMTP設定の存在 + 必須項目妥当性で仮判定（実送信疎通は未実装）
- maintenance: 環境変数で制御

## 受け入れ条件
- `/status` ページで、アプリ/API/DB/認証/メール/メンテナンスの状態を確認できる
- `normal / warning / outage / maintenance` を視認しやすく表示
- 最終更新時刻を表示
- 公開APIと管理APIで情報量を分離
- README と本Issue文書、Qiita下書きが更新されている

## 今後の引き継ぎ
- ステータス履歴管理（時系列）
- 外部監視連携（Datadog, PagerDutyなど）
- 管理画面でのメンテナンス切り替えUI
